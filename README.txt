残高予報 v1.3.0
=================

家計簿、現金・銀行・電子マネーの保有残高、クレジットカード未払、ローン、将来残高を一体管理するローカル完結型PWAです。

公開に必要なファイル
--------------------
次の7ファイルを同じ公開ディレクトリへ置いてください。

- index.html
- manifest.json
- sw.js
- icon.svg
- icon-192.png
- icon-512.png
- apple-touch-icon-180.png

利用条件
--------
- HTTPSで公開すること
- SafariまたはChromeの通常URLとして開くこと
- ChatGPTの添付HTMLプレビューやfile://では、PWA・Service Workerとして動作しません

主要画面
--------
1. 家計簿
   - 保有残高、カード未払、短期実質残高、純金融残高
   - 月間支出・収入・収支・予算
   - 前月・3カ月平均比較
   - 固定費・変動費・特別支出
   - カテゴリ別・支払方法別集計
   - 検索、絞込、並べ替え、ページ分割

2. 予報
   - 今すぐ使える金額
   - 今月使える残額
   - 1日あたり目安
   - 口座別不足、危険日、残高推移

3. 予定
   - 月間カレンダー
   - 入出金・カード請求・ローン・振替の一覧

4. 登録
   - 銀行口座
   - 現金・財布
   - 電子マネー残高
   - クレジットカード
   - 定期入出金
   - リボ・ローン
   - 一度だけの入出金
   - 資金移動・カードチャージ

5. 設定
   - 月予算、カテゴリ予算、繰越方式
   - 定型入力
   - 営業日・独自休業日
   - 自動スナップショット
   - 年度アーカイブ
   - 文字拡大
   - バックアップ・復元

現金の扱い
----------
現金は画面上で「口座」とは呼びません。「現金・財布」と表示します。
内部では残高計算のため銀行・財布・電子マネーを同じ保有単位として扱います。

- 銀行から財布へ引出：銀行減、財布増、支出0、総保有残高不変
- 財布から支払：財布減、家計簿支出増、総保有残高減
- 銀行から電子マネーへチャージ：銀行減、電子マネー増、総保有残高不変
- カードから電子マネーへチャージ：電子マネー増、カード未払増、短期実質残高不変

クレジットカードの扱い
----------------------
カード利用日には、銀行・現金の保有残高を減らしません。

- 家計簿：利用日に支出を計上
- カード未払：利用額を増加
- 残高予報：締日・支払日から将来引落を生成
- 引落日：口座残高を減少
- 引落は「決済・返済」として家計簿支出から除外し、二重計上を防止

カード会社固有の処理は完全には推測できないため、確定請求額で上書きできます。

保存とバックアップ
------------------
- 通常保存：ブラウザlocalStorage
- 自動スナップショット：最大4世代
- 手動バックアップ：JSON
- 月別家計簿：CSV
- 年度アーカイブ：JSON

入力データはアプリから外部送信しません。

注意
----
- 金融機関・カード会社の公式残高・請求額ではありません
- 未登録取引、処理時刻、会社独自の締め処理、手数料規約により実額と差が出ます
- 重要な支払いは公式明細と照合してください
- iPhone実機Safari、ホーム画面追加後、Android実機は公開URL上で最終確認が必要です
